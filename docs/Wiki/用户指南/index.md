## 核心概念

### CC-Workflow 是什么？

CC-Workflow 是一个以 Claude Code 为核心的团队协作工作流系统。它实现了从需求到上线的全流程自动化，让团队成员只需专注于三件事：**提需求、做评审、反馈意见**。

### 核心原则

| 角色 | 职责 |
|------|------|
| **团队成员** | 提需求、做评审、反馈意见 |
| **Claude Code** | 需求编写、技术方案、计划生成、开发执行、测试用例、维基更新 |

### 15 阶段流程

CC-Workflow 将软件开发分为 15 个阶段（P1-P15），分为四个阶段组：

```mermaid
flowchart LR
    subgraph 规划[P1-P4 规划阶段]
        P1((P1)) --> P2((P2)) --> P3((P3)) --> P4((P4))
    end

    subgraph 准备[P5-P8 准备阶段]
        P5((P5)) --> P6((P6)) --> P7((P7)) --> P8((P8))
    end

    subgraph 执行[P9-P12 执行阶段]
        P9((P9)) --> P10((P10)) --> P11((P11)) --> P12((P12))
    end

    subgraph 收尾[P13-P15 收尾阶段]
        P13((P13)) --> P14((P14)) --> P15((P15))
    end

    规划 --> 准备 --> 执行 --> 收尾
```

详细的阶段说明请参阅 [流程概述](./流程概述.md)。
