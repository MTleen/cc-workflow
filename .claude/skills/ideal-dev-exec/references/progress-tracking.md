# 进度跟踪

## 任务状态更新

### 状态定义

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| `[ ]` | 未开始 | 任务初始化 |
| `[x]` | 已完成 | 验证通过 |

### 更新时机

1. 任务开始时：创建任务记录
2. 每个子步骤完成时：更新步骤状态
3. 任务完成时：标记为 `[x]`

## 执行报告格式

### 任务开始报告

```markdown
## 开始任务: {任务编号}

**目标**: {任务目标}

**预计步骤**: {步骤数量}
```

### 任务进度报告

```markdown
## 任务进度: {任务编号}

**已完成步骤**: {n}/{total}

**当前步骤**: {当前步骤描述}

**遇到问题**: {如有}
```

### 任务完成报告

```markdown
## 完成任务: {任务编号}

**目标**: {任务目标}

**完成内容**:
- {完成的步骤1}
- {完成的步骤2}

**验证结果**: {通过/失败}

**耗时**: {时间}
```

## 异常处理流程

### 遇到阻塞

1. 记录阻塞原因
2. 尝试自行解决（5分钟内）
3. 无法解决时：
   - 保存当前进度
   - 报告阻塞问题
   - 等待人工介入

### 测试失败

1. 分析失败原因
2. 如果是测试问题：
   - 修复测试
   - 重新运行
3. 如果是代码问题：
   - 回到 GREEN 阶段
   - 修复代码
   - 重新运行

### 环境问题

1. 记录环境信息
2. 尝试标准修复方法
3. 无法解决时报告
