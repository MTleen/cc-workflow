# 用户文档写作指南

## 概述

用户文档面向**最终用户**和**产品经理**，目标是帮助用户快速上手并熟练使用产品。

### 目标读者画像

| 读者类型 | 特点 | 需求 |
|----------|------|------|
| 新用户 | 无使用经验 | 快速上手，理解核心概念 |
| 普通用户 | 有基础使用经验 | 解决问题，学习高级功能 |
| 产品经理 | 评估/决策 | 了解功能范围、限制 |

---

## 文档集结构

用户文档不是单文件，而是有逻辑结构的一组文件：

```
用户文档/
├── README.md          # 文档索引/首页
├── 快速开始.md        # 5分钟上手
├── 使用指南.md        # 详细功能说明
├── 常见问题.md        # FAQ
└── 更新日志.md        # 版本历史
```

---

## 各文档写作指南

### README.md（文档索引）

**目的**：作为文档入口，帮助用户快速找到所需内容。

**必备内容**：
1. 产品简介（1-2 句话）
2. 文档导航（链接到其他文档）
3. 快速链接（常见任务入口）

**示例结构**：
```markdown
# {产品名称} 用户文档

## 简介

{产品} 是一个 {一句话描述}，帮助 {目标用户} {解决什么问题}。

## 文档导航

| 文档 | 说明 |
|------|------|
| [快速开始](快速开始.md) | 5分钟上手教程 |
| [使用指南](使用指南.md) | 详细功能说明 |
| [常见问题](常见问题.md) | FAQ 解答 |
| [更新日志](更新日志.md) | 版本历史 |

## 常见任务

- [如何完成 XXX？](使用指南.md#xxx)
- [如何配置 YYY？](使用指南.md#yyy)
```

---

### 快速开始.md

**目的**：让新用户在 **5 分钟内** 完成核心流程，获得即时反馈。

**写作原则**：
- 步骤不超过 5 步
- 只包含最核心的功能
- 每步都有预期结果
- 避免可选步骤和分支

**示例结构**：
```markdown
# 快速开始

## 前置条件

- {必备条件 1}
- {必备条件 2}

## 步骤

### 1. {第一步标题}

{简短说明}

```bash
{具体命令或操作}
```

**预期结果**：{描述成功后的状态}

### 2. {第二步标题}

...

## 下一步

- 了解更多功能：[使用指南](使用指南.md)
- 遇到问题：[常见问题](常见问题.md)
```

---

### 使用指南.md

**目的**：详细说明所有功能，作为用户的完整参考手册。

**写作原则**：
- 按功能模块组织
- 每个功能包含：说明、步骤、预期结果
- 复杂操作配图或流程图
- 提供实际使用场景

**示例结构**：
```markdown
# 使用指南

## 功能概览

{功能列表或功能架构图}

## {模块 1 标题}

### {功能 1.1}

**说明**：{功能描述}

**使用场景**：{什么时候使用}

**操作步骤**：
1. {步骤 1}
2. {步骤 2}

**预期结果**：{描述成功后的状态}

> 💡 **提示**：{使用技巧或最佳实践}

---

## {模块 2 标题}

...
```

### 常见问题.md

**目的**：解答用户最常遇到的问题，减少支持成本。

**写作原则**：
- 问题来源于真实用户反馈
- 按问题类型分类
- 回答简洁，包含解决方案
- 链接到相关文档

**示例结构**：
```markdown
# 常见问题 (FAQ)

## 安装与配置

### Q: {安装相关问题}？

**A**: {简洁回答}

{如需详细步骤，提供链接}

---

## 使用问题

### Q: {使用相关问题}？

**A**: {简洁回答}

---

## 故障排除

### Q: {报错或异常情况}？

**A**: {可能原因和解决方案}

1. 检查 {项目 1}
2. 确认 {项目 2}
3. 如果仍有问题，{下一步操作}
```

---

### 更新日志.md

**目的**：记录版本变化，让用户了解新功能和变更。

**写作原则**：
- 按版本倒序排列（最新在前）
- 使用清晰的分类（新增/改进/修复/移除）
- 包含日期和版本号

**示例结构**：
```markdown
# 更新日志

所有重要的变更都会记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/)。

## [Unreleased]

### 新增
- {即将发布的新功能}

## [1.2.0] - 2026-02-21

### 新增
- {新功能描述} (#123)

### 改进
- {改进描述}

### 修复
- {Bug 修复描述}

### 移除
- {移除的功能}

## [1.1.0] - 2026-02-15

...
```
